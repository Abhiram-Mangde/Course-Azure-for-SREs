# Multi-Factor Authentication (MFA) in Azure

## What is MFA?
Multi-Factor Authentication (MFA) is a security feature in Azure that requires users to prove their identity using two or more verification methods before they can access resources. Instead of just typing a password, users must provide an additional factor like a phone code, fingerprint, or app approval.

*Think of it as a double lock on your account:*
- First lock = your password
- Second lock = something else only you have (phone, app, biometric)

## How Does MFA Work?
*When a user tries to sign in:*
1. They enter their username and password (first factor).
2. Azure asks for another verification (second factor), such as:
    - A code sent via SMS or phone call
    - A push notification in the Microsoft Authenticator app
    - A one-time password (OTP) generated by an app
    - Biometric verification (fingerprint/face recognition)
3. Only if both checks succeed, the user gets access.

## How to Use MFA in Azure
1. Enable MFA in Azure Active Directory (Azure AD).
- Admins can enforce MFA for all users or specific groups.
- Conditional Access policies can require MFA only in certain situations (e.g., logging in from outside the office network).
2. Register MFA methods:
- Users set up their preferred verification method (Authenticator app, SMS, phone call, etc.).
3. Sign in with MFA:
- After entering the password, users complete the second step using their chosen method.

## Details & Options
Azure MFA supports multiple verification methods:
- Microsoft Authenticator app (recommended, fastest, most secure)
- SMS or phone call (easy but less secure)
- Hardware tokens (for high-security environments)
- Biometric authentication (if supported by device)

Admins can configure policies like:
- Require MFA only for sensitive apps
- Block access if MFA is not completed
- Allow trusted devices to skip MFA for convenience

## Advantages of MFA
- **Stronger Security**: Protects against stolen or weak passwords.
- **Defense Against Phishing**: Even if someone knows your password, they can’t log in without the second factor.
- **Flexibility**: Users can choose the method that works best for them.
- **Compliance**: Helps meet security standards and regulations.
- **Conditional Access**: MFA can be applied only when risk is high (e.g., unusual login location).

## Example
*Imagine you’re an SRE logging into the Azure Portal:*
1. You type your username and password.
2. Instantly, a notification pops up on your phone via the Authenticator app.
3. You tap Approve.
4. You’re in — safe and secure.

Without MFA, if someone stole your password, they could log in. With MFA, they’d also need your phone or fingerprint, which makes attacks much harder.
